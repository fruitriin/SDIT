# Phase 22.1: タブのドラッグ切り出し（Chrome-like UX）

## 要望

複数タブがあるときに、サイドバーのタブをカーソルドラッグしてウィンドウ外に出すと、独立したウィンドウに切り出したい（Chrome-like UX）。

## 現状

| 項目 | 状態 |
|---|---|
| サイドバー描画 | wgpu `CellPipeline`（カスタム描画） |
| サイドバー内ドラッグ並び替え | 実装済み（`sessions.swap()`） |
| DetachSession（キーバインド/メニュー） | 実装済み（`Cmd+Shift+N`、右クリックメニュー） |
| ウィンドウ外へのドラッグ切り出し | **未実装** |
| ウィンドウへのドラッグ合体 | **未実装** |

## アプローチ比較

### A案: 現行 wgpu サイドバーの拡張（推奨）

サイドバー内ドラッグ中にカーソルがウィンドウ外に出たことを検出し、`DetachSession` を発動する。

**実装方針:**
1. `CursorMoved` でドラッグ中にカーソル座標がウィンドウ領域外かを判定
2. ウィンドウ外に出たら `detach_session_to_new_window()` を呼び出し
3. 新ウィンドウをカーソル位置に配置（`window.set_outer_position()`）
4. （将来）ドラッグ中に別の SDIT ウィンドウ上にドロップしたら合体

**メリット:**
- 縦タブの設計哲学を維持
- 既存の `detach_session_to_new_window()` をそのまま再利用
- クロスプラットフォーム対応可能
- 変更範囲が小さい（`event_loop.rs` のドラッグ処理のみ）

**デメリット:**
- ドラッグ中のビジュアルフィードバック（ゴーストタブ等）は自前実装が必要
- ウィンドウ間ドラッグ合体は別フェーズで追加実装が必要

### B案: macOS ネイティブタブ（NSWindow tabbing）

macOS の `NSWindow` タブ機能を objc2 で直接操作し、OS 標準のタブ UI を使う。

**メリット:**
- ドラッグ&ドロップが OS 標準で動く（切り出し・合体の両方）
- macOS ユーザーにとって自然な操作感

**デメリット:**
- NSWindow tabbing は**横タブ**。SDITの「縦タブ」コンセプトと矛盾する
- SDIファーストの哲学に反する（macOS がタブバーを常時表示する制御をする）
- winit 0.30 が NSWindow tabbing API を公開していないため、objc2 で直接叩く必要がある
- macOS 専用。クロスプラットフォーム対応不可
- 現在のサイドバー実装（wgpu 描画、セッション管理、リネーム機能等）を全面的に書き換える必要がある
- wgpu の描画領域と NSWindow タブの連携が複雑

### C案: ハイブリッド（macOS タブ + 縦サイドバー併存）

NSWindow tabbing をドラッグ切り出し/合体の「裏方」としてのみ使い、表示は縦サイドバーのままにする。

**デメリット:**
- 2つのタブ管理システムの同期が必要で複雑性が高い
- NSWindow のタブバーを非表示にしつつタブ機能だけ使うのは macOS API の想定外

## 推奨: A案

SDIT の設計哲学（SDIファースト、縦タブ）を維持しつつ、最小の変更で Chrome-like UX を実現できる。

## 実装計画（A案）

### Step 1: ドラッグ切り出し
1. `event_loop.rs` のサイドバードラッグ処理に「ウィンドウ外判定」を追加
2. カーソルがウィンドウ領域外に出たら `detach_session_to_new_window()` を呼ぶ
3. 新ウィンドウの位置をカーソル座標に設定

### Step 2: ビジュアルフィードバック（オプション）
- ドラッグ中のタブ行をハイライト表示
- ウィンドウ外に出る直前の閾値でカーソルを変更

### Step 3: ドラッグ合体（将来フェーズ）
- 別の SDIT ウィンドウ上にドロップしたらセッションを合体
- これは別の計画ファイルとして分離する

## 変更対象

- `crates/sdit/src/event_loop.rs` — サイドバードラッグ中の CursorMoved / CursorLeft 処理
- `crates/sdit/src/window_ops.rs` — `detach_session_to_new_window()` に位置指定パラメータ追加

## セキュリティ影響

なし
