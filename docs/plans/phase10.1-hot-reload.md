# Phase 10.1: 設定Hot Reload

**概要**: 設定ファイル変更時に自動的にリロードする機能を実装する。

## タスク

| タスク | 詳細 | 変更先クレート |
|---|---|---|
| ファイル監視 | `notify` クレートで `sdit.toml` の変更を監視 | sdit |
| 差分適用 | フォント・カラー・キーバインドの変更をリロード | sdit, sdit-config |

## 依存関係

Phase 9.2（キーバインドカスタマイズ。メニューとショートカットの一元管理が前提）

## 新規依存クレート

`notify`

## セキュリティ考慮事項

- 設定ファイル監視。TOCTOU競合のリスク → アトミック読み込みで対応
