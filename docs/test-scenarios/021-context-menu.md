# テストシナリオ 021: 右クリックコンテキストメニュー

## 概要

ターミナル領域とサイドバー領域での右クリックによるコンテキストメニューの表示・動作を確認する。

## 前提条件

- macOS 環境であること（本機能は `#[cfg(target_os = "macos")]` で macOS 専用）
- SDIT ウィンドウが起動していること
- Accessibility 権限が付与されていること

## テスト手順

### 021-1: ターミナル領域の右クリックメニュー表示

1. SDIT ウィンドウのターミナル領域（サイドバー以外）で右クリックする
2. **期待**: コンテキストメニューが表示される
3. **期待**: メニューには以下の項目が含まれる:
   - Copy
   - Paste
   - （セパレータ）
   - Select All
   - （セパレータ）
   - Search…
4. Escape キーまたはメニュー外クリックでメニューを閉じる
5. **期待**: メニューが閉じ、ターミナル操作が継続できる

### 021-2: ターミナル領域 — Copy 動作

1. ターミナルに何かテキストを表示する（例: `echo hello`）
2. マウスでテキスト「hello」を選択する
3. ターミナル領域で右クリックし、「Copy」を選択する
4. **期待**: 選択したテキストがクリップボードにコピーされる
5. 別のアプリにペーストして「hello」が取得できることを確認する

### 021-3: ターミナル領域 — Paste 動作

1. クリップボードに何かテキストをコピーしておく（例: `echo test\n`）
2. ターミナル領域で右クリックし、「Paste」を選択する
3. **期待**: クリップボードのテキストがターミナルに貼り付けられる

### 021-4: ターミナル領域 — Select All 動作

1. ターミナルで適当なコマンドを実行してテキストを表示する
2. ターミナル領域で右クリックし、「Select All」を選択する
3. **期待**: 画面上のテキストが全て選択状態になる（テキスト選択ハイライトが表示される）

### 021-5: ターミナル領域 — Search 動作

1. ターミナルで適当なコマンドを実行してテキストを表示する
2. ターミナル領域で右クリックし、「Search…」を選択する
3. **期待**: 検索バーが表示される（Phase 9.1 実装後に確認）

### 021-6: サイドバー領域の右クリックメニュー表示

前提: セッションが 2 つ以上あり、サイドバーが表示されていること

1. Cmd+T でセッションを追加してサイドバーを表示する
2. サイドバーの任意のセッション項目上で右クリックする
3. **期待**: サイドバー用コンテキストメニューが表示される
4. **期待**: メニューには以下の項目が含まれる:
   - Close Session
   - Move to New Window
5. **期待**: 右クリックしたセッションがアクティブになる（アクティブ状態ハイライトが表示される）

### 021-7: サイドバー — Close Session 動作

前提: セッションが 2 つ以上あること

1. サイドバーの非アクティブなセッション項目を右クリックする
2. 「Close Session」を選択する
3. **期待**: そのセッションが閉じられる
4. **期待**: サイドバーのセッション数が 1 つ減る
5. セッションが 1 つになった場合は **期待**: サイドバーが非表示になる（SDI 状態へ遷移）

### 021-8: サイドバー — Move to New Window 動作

前提: セッションが 2 つ以上あること

1. サイドバーのセッション項目を右クリックする
2. 「Move to New Window」を選択する
3. **期待**: そのセッションが新しい独立ウィンドウとして開く
4. **期待**: 元のウィンドウのセッション数が 1 つ減る
5. **期待**: PTY セッション（実行中のプロセス）は切断されずに維持される
6. 新しいウィンドウで `echo hello` を実行し、レスポンスが返ることを確認する

### 021-9: 複数ウィンドウでの右クリック独立動作

1. Cmd+N で 2 つのウィンドウを開く
2. 片方のウィンドウのターミナル領域で右クリックしてメニューを表示する
3. **期待**: メニューがそのウィンドウの位置付近に表示される
4. **期待**: もう片方のウィンドウは影響を受けない（フォーカスが変わらない）

### 021-10: アクション共有マップの不変性確認

1. ターミナル領域で右クリックしてメニューを閉じる（Copy/Paste/Select All/Search が追加）
2. サイドバー領域で右クリックしてメニューを閉じる（CloseSession/DetachSession が追加）
3. メニューバーの Edit > Copy（Cmd+C）を実行する
4. **期待**: メニューバーの Copy が正常に動作する（コンテキストメニューのアクションが上書きしない）

## 自動テスト

`cargo test` で以下のユニットテストが実行される:

- `menu::tests::context_menu_actions_are_defined` — コンテキストメニューの Action バリアント存在確認
- `menu::tests::menu_actions_are_defined` — メニューバーの Action バリアント存在確認（コンテキストメニューとの共存確認）

## 実装メモ

- コンテキストメニュー表示は `muda::ContextMenu::show_context_menu_for_nsview()` を使用
- `MenuEvent` ハンドラは `Arc<Mutex<HashMap<MenuId, Action>>>` を共有しており、コンテキストメニューの ID を `extend()` で追加する
- `menu.rs` ファイルのみ `#![allow(unsafe_code)]` を付与し、unsafe スコープを限定している
- サイドバー領域の判定は `origin_x < sidebar_width_px` で行う
