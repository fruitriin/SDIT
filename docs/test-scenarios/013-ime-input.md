# テストシナリオ 013: IME入力

## 概要

macOS IME（日本語入力）の基本動作を確認する。

## 前提条件

- macOS の入力ソースに「日本語 - ローマ字入力」が有効であること
- SDIT ウィンドウがフォーカスされていること

## テスト手順

### 013-1: 基本的な日本語入力

1. SDIT ウィンドウでテキスト入力可能な状態（シェルプロンプト等）にする
2. macOS の入力ソースを日本語に切り替える
3. 「konnichiha」とタイプする
4. **期待**: プリエディット（変換中テキスト）がカーソル位置にインライン表示される
5. **期待**: プリエディットテキストは通常の背景より少し明るい背景色で表示される
6. Enter で確定する
7. **期待**: 確定したテキスト「こんにちは」がターミナルに表示される

### 013-2: 変換候補の選択

1. 「kanji」とタイプし、スペースキーで変換する
2. **期待**: 変換候補ウィンドウがカーソル位置付近に表示される
3. 矢印キーで候補を選択し、Enter で確定する
4. **期待**: 選択した候補がターミナルに表示される

### 013-3: 変換キャンセル

1. 「nihongo」とタイプする
2. **期待**: プリエディットが表示される
3. Escape キーを押す
4. **期待**: プリエディットが消え、入力がキャンセルされる

### 013-4: 全角文字の幅

1. 「あいう」と入力して確定する
2. **期待**: 各文字が2セル幅で表示される（半角文字の2倍の幅）

### 013-5: 複数ウィンドウでの IME

1. Cmd+N で新しいウィンドウを開く
2. 新しいウィンドウで日本語入力を行う
3. **期待**: 各ウィンドウで独立に IME 入力できる
4. **期待**: 変換候補ウィンドウがアクティブウィンドウのカーソル位置に表示される

## 自動テスト

`cargo test` で以下のユニットテストが実行される:

- `app::tests::preedit_state_*` — PreeditState の構築・更新
- `app::tests::ime_commit_*` — IME Commit テキストのバイト変換・サニタイズ
- `render::tests::*_width` — char_cell_width の全角/半角判定
